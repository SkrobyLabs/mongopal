# Schema analysis

## What it does

Infers the shape of a collection by sampling documents and reporting each field's observed
types and how often they occur (for example, a field that is an ObjectId in every sampled
document versus one that is sometimes a string and sometimes null). Nested structures are
surfaced, and the inferred schema can be exported as JSON. A companion collection profile
reports aggregate statistics — document count, average document size, field count, nesting
depth — for spotting bloated documents and schema sprawl.

## Why it exists

MongoDB collections are schemaless, so the only way to understand unfamiliar data is to
look at it — tediously, document by document. Schema inference answers "what is actually in
here?" in one pass, and the same inferred schema powers autocomplete and type coercion in
the [query modes](query-modes.md) and grounds the [AI assistant](ai-query-assistant.md).
The profile statistics turn that understanding into capacity and performance signals.

## Constraints & trade-offs

- Inference is sampling-based, not an exhaustive scan, so it stays fast on large
  collections at the cost of possibly missing rare field variations. The sample size is
  configurable.
- It reports type frequency per field rather than a formal, constraint-checked schema.
