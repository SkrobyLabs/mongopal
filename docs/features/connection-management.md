# Connection management & credential security

## What it does

Stores and organizes MongoDB connections and keeps their secrets safe. Connections can be
defined by URI or by a multi-tab form, organized into nested drag-and-drop
[folders](../glossary.md), starred as [favorites](../glossary.md), duplicated to spin up
environment variants, and tested before saving. Beyond username/password, connections
support the range of MongoDB auth mechanisms plus SSH tunneling, TLS/mTLS, and a SOCKS
proxy, along with driver-level tuning: connection-pool sizing, read preference, write
concern, network compression, and a custom application name. Connections can be exported
and imported as encrypted bundles to move between machines or share with teammates.

## Why it exists

Anyone working across several MongoDB deployments needs a safe, organized home for many
sets of credentials — without pasting passwords into shared config files or leaking them
into logs and process listings. MongoPal treats credential confidentiality as a core
guarantee rather than an afterthought, and folds the operational knobs (TLS, SSH, driver
tuning) into the same place so a secured remote connection is a matter of filling in a form.

## How it protects secrets

Each connection is stored on disk encrypted with AES-256-GCM, and the encryption key lives
in the OS keyring rather than beside the data. Passwords are stripped from anything
displayed or logged, and are handed to subprocesses through environment variables so they
never appear in `ps` output. Revealing a stored password, or exporting credentials,
requires OS-level authentication (biometric or system password) with a short grace period,
after which access re-locks.

## Safety rails for destructive operations

Per connection, drop and clear operations can be gated behind a deliberate delay and a
typed confirmation, so an accidental click on a production connection cannot instantly
destroy data.

## Constraints & trade-offs

- On a machine without a working keyring, encryption falls back to an ephemeral key: data
  is still encrypted but the files stop being portable across restarts.
- The older saved-connection format (keyring password + plain URI) is still supported for
  backward compatibility, so two storage paths coexist.

See [decision 2](../decisions.md) for the storage model rationale.
