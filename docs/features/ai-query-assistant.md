# AI query assistant

## What it does

Turns a plain-language description ("the ten most recent orders over $100") into a runnable
query. The user types intent, picks the target dialect (native MongoDB or SQL), and the
assistant returns a generated query plus a short explanation, which can be copied or dropped
straight into the editor. Each generation is a fresh, one-shot request — there is no chat
history or multi-turn conversation — and the panel reports the token counts for the call so
cost stays visible.

## Why it exists

Writing MongoDB queries — especially aggregations — has a real learning curve, and even
fluent users lose time on syntax for one-off questions. Describing the goal in words removes
that friction, while the assistant grounds its output in the collection's actual
[inferred schema](schema-analysis.md) so the generated query references real fields with the
right types rather than guessing.

## Privacy model

This is the feature's defining constraint. Only the inferred schema — field names, types, and
how often each appears — plus the collection name and the user's prompt are sent to the model.
**Document values, database names, connection URIs, and credentials are never sent.** Field
names are sanitized and fenced before being placed in the prompt to guard against injection.
The Anthropic API key is stored in the OS keyring (or read from an environment variable for
CI) with **no file fallback at all**; the settings panel reports its status without ever
displaying it. See [decision 3](../decisions.md).

## Constraints & trade-offs

- Anthropic is the only provider today, though the backend is written behind a provider
  interface so another could be added. The user chooses between a higher-quality and a faster
  model tier.
- The assistant is optional and off until an API key is configured.
- It generates queries, it does not execute them or return data — the user reviews and runs
  the result themselves.
- Best on straightforward filters and aggregations; complex nested pipelines may need manual
  refinement.
